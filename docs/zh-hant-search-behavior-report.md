# 대만·번체 사용자는 구글에서 어떻게 검색하는가 — 실측 보고서

> 목적: `/zh-hant/` 번체 포스팅의 제목·H2·구조를 **추측이 아니라 실측**으로 설계하기 위함.
> 방법: ① rakko `headline`(상위페이지 H1~H3 추출) ② rakko `question-search`(질문DB) ③ **구글 번체 실검색**(WebSearch, 4개 쿼리) 교차검증.
> 대상 키워드: `德州撲克規則`, `德州撲克牌型大小`, `德州撲克怎麼玩`, `德州撲克 位置 攻略` 등.
> 작성: 2026-07-22

---

## 0. 한 줄 결론
대만 번체 검색은 **영어식 자연어 질문형이 아니라 "키워드·명사형"이 지배**한다. 제목은 `【태그】+시간약속+키워드스택+감정훅+브랜드` 공식이 이기고, 질문은 본문이 아니라 **FAQ 블록**에 격리된다. 따라서 **EN 마스터의 질문형 H2를 그대로 번역·이식하면 안 된다.**

---

## 1. 검색 "형태" — 키워드형 지배, 질문형 희소

- **rakko 질문DB에서 `德州撲克` 질문 = 0건.** 영어권('what beats a flush', 'how to play poker')과 달리, 번체는 완전한 자연어 질문으로 검색하는 비중이 낮다.
- 실제 검색은 **명사·키워드 조합**: `德州撲克`+`規則`/`牌型大小`/`教學`/`位置`/`玩法`/`術語`.
- **단, 짧은 `怎麼~?` (how) 단편은 실제로 쓰임** — `德州撲克怎麼玩`, `大小怎麼看`, `怎麼比大小`. 이는 제목·H2 훅으로 활용 가능.
- **`是什麼`(what is)는 약함** — 정의 1줄 용도.

> **시사점**: 본문 H2는 명사형 토픽으로, 질문은 FAQ 블록에만. `怎麼~?` 훅은 제목/소제목에 제한적으로.

## 2. 실제 검색어 패턴 (키워드 조합)

| 축 | 실제 조합 예 | 인텐트 |
|---|---|---|
| 규칙/입문 | 德州撲克規則 · 玩法 · 新手教學 · 入門 | 초보 정보형 |
| 족보 | 德州撲克牌型大小 · 大小順序 · 同花比大小 | 정보형(최대 수요) |
| 방법 | 德州撲克怎麼玩 · 大小怎麼看 · 怎麼比 | how-형(짧은) |
| 전략 | 德州撲克 位置 攻略 · 起手牌 · 技巧 · 策略 | 전략형(`攻略`가 핵심 수식어) |
| 용어 | 德州撲克 術語 | 참조형 |

- 수식어 위계: **`攻略`·`教學`·`技巧`(고인텐트) > `規則`·`玩法`·`大小`(정보) > `是什麼`(약)**.
- 롱테일은 **위치변형**으로: `德州撲克+X`(뒤) 뿐 아니라 앞·중간도 확인 필요.

## 3. 상위글 제목(H1/타이틀) 공식 — 구글 실측

실제 1페이지 제목에서 반복 관찰된 5요소:
1. **【태그】 대괄호 훅** — `【新手教學】` · `【圖解】` · `【德州撲克位置】`
2. **시간·원스톱 약속** — `5分鐘搞懂` · `3分鐘了解` · `一篇` · `一次看/一次懂` · `這一篇全都包`
3. **키워드 스택**(구분자 `、｜／`) — `玩法/位置/規則/大小`, `位置、規則、術語一次懂`
4. **감정 훅** — `位置就是一切！` · `看這篇就夠了` · `我要當C位` · `還學不會分辨？`
5. **브랜드 접미** — `- 包手撲克` · `｜錢盈娛樂城` · `- Monsterstack`

실측 제목 원문:
- `【新手教學】5分鐘搞懂德州撲克規則玩法！｜錢盈娛樂城`
- `德州撲克新手入門教學，玩法/位置/規則/大小一篇查理教會你！`
- `德州撲克超新手教學，規則？大小？實用技巧？這一篇全都包啦！`
- `【德州撲克位置】位置就是一切！我要當C位 - 贏家娛樂城`
- `德州撲克的位置怎麼看？3分鐘了解各位置優劣勢`

> **키워드 2~3회 반복(스터핑)은 관대**하게 허용됨. 질문형 풀문장 제목은 드묾.

## 4. 상위글 본문 구조(H2/H3) — rakko headline 실측

공통 골격(taiwanrounders·Natural8·yangbear·sixpoker 등):
- **H2 = 키워드 박은 명사형 토픽**: `德州撲克玩法簡介` · `基本規則與遊戲流程` · `手牌大小排名一覽（從大到小）` · `德州撲克實用技巧`.
- **족보 = 번호 매긴 리스트**(H3): `1. 皇家同花順（Royal Flush）` … `10. 高牌（High Card）`.
- **베팅 라운드 = 분해**: `翻牌前(Pre-Flop)` / `翻牌(Flop)` / `轉牌(Turn)` / `河牌(River)`.
- **질문은 FAQ 블록에 격리**: `常見問題 FAQ` / `德州撲克規則-問與答` 섹션 아래에 개별 질문(H3): `那如果兩個人牌型一樣的時候呢?`, `Q: 如何在線上玩德州撲克遊戲?`.
- **용어 中(英) 병기 표준**: `皇家同花順(Royal Flush)`, `莊家按鈕位(The Dealer Button)`, `盲抓(The Straddle)`.
- **마무리**: `總結` / `結語` / `教練提醒`.
- **분량**: 상위 평균 **6,510자**(`德州撲克規則` 기준), 상위글 5,500~19,655자 → **롱폼 원스톱**이 이김.

## 6. 우리 번체 포스트에 대한 시사점 (설계 규칙)
1. **제목**: §3 공식 적용(【태그】+시간약속+키워드스택+감정훅). 질문 풀문장·`是什麼` 금지, `怎麼~?` 훅만 제한적.
2. **본문 H2**: 명사형 토픽 + 키워드 연속(끊김 없이). 질문은 `常見問題 FAQ` 블록에만.
3. **족보/라운드**: 번호 리스트 + 中(英) 병기 표준 준수(대만 독자 기대치).
4. **분량**: 롱폼 원스톱 지향(얇은 제휴글 대비 깊이로 승부).
5. **수식어**: `攻略/教學/技巧`를 전략글 축으로.
6. **편당**: `rakko headline`로 그 키워드 상위 골격 재확인 후 착수(추측 금지).

## 7. 한계·후속
- WebSearch는 US 라우팅 → 대만 로컬 순위와 미세차이 가능(경쟁 "집합"은 신뢰, 정확한 순위는 아님).

# SEO 카피라이팅 가이드 — 호기심 훅 + 키워드 하이브리드

> 이 사이트 모든 포스트의 제목(`title`/`seoTitle`) · 설명(`description`/`desc`) · 첫 단락(`tldr`) 작성 원칙.
> 함께 보는 문서: `PROJECT.md`, `PILLAR-CLUSTER-ROADMAP.md`.

---

## 🚨 최우선 — 글쓰기 전 경쟁 분석 (절대 생략 금지)

**타깃 키워드로 구글 검색 → 상위 10개 경쟁 글 분석 → 그보다 경쟁력 있는 글을 만든다.**
메타·본문·이미지를 쓰기 **전에** 아래를 끝낸다. 이 단계 없이 작성 시작 = 금지.

| 순서 | 할 일 | 산출물 |
|:--:|---|---|
| 1 | 타깃 키워드 1~3개 확정 (언어별 검색량·롱테일·사용자 의도) | 키워드 리스트 |
| 2 | 시크릿 모드로 키워드 그대로 검색 | SERP URL 목록 |
| 3 | 상위 10개(광고 제외, 블로그·가이드) 열어보기 | 경쟁 글 체크리스트 |
| 4 | H2 개수·분량·FAQ·표·예시·도표 비교 | "우리가 더 줄 것" 3가지 |
| 5 | 차별 포인트 정하기 (실전 예시, 빠진 질문, 더 나은 표, 다국어 동시 발행) | 글 설계 1페이지 |
| 6 | 설계대로 한국어 초안 → 영어 마스터 → 번역 발행 | 배포 |

### 경쟁 분석 시 반드시 메모할 것
- 상위 글 **평균 H2 개수 / 대략 분량** (짧으면 깊게, 길면 핵심만 압축)
- **빠진 질문** (People Also Ask·FAQ·댓글엔 있는데 본문에 없는 것)
- **우리만 넣을 것**: 실전 예시·도표, 입문자 맞춤 설명, 클러스터 내부링크
- **이기기 어려운 것**: 대형 사이트·동영상 독점 주제 → 롱테일·각도 변경

> **한 줄 원칙: "정보를 더 많이"가 아니라 "검색 의도를 더 정확히 + 클릭·체류를 더 잘" 충족하는 글.**
> 언어마다 검색량·경쟁이 다르니 **각 언어 SERP도 따로 확인** 권장.

## 🧭 핵심 철학

**구글 검색 결과의 제목·설명 = 유튜브 썸네일.** 글이 좋아도 메타가 평범하면 클릭이 안 일어난다.

- 포커 SERP 경쟁 글 대부분은 "OO 뜻 / OO 정리 / OO 가이드 / Ultimate Guide" 같은 **딱딱한 정보형**.
- 살아남으려면 **호기심·공감·감정 훅**으로 시선을 잡되, 알고리즘이 관련성을 알도록 **핵심 키워드**도 넣는다.
- 즉, **호기심 훅 + 핵심 키워드 = 하이브리드**가 정답.

## 📐 검증된 공식

### Title
```
[호기심·자극·공감 훅] — [핵심 키워드 명사 2~3개]
```
- 길이: 한국어 약 28~34자 / 영어 약 50~60자 (모바일 SERP 안전선).
- 구분자는 **em dash (—)** 통일. 끝 단어는 **명사로 종결**.
- "완벽 정리 / 총정리 / 모든 것 / Ultimate Guide / All About" **금지**.
- 본문 H1과 100% 중복 금지 (검색용·본문용은 보완 관계).

### Description
```
[첫 문장: 검색자의 고민·증상에 공감]. [둘째 문장: 무엇을 알 수 있는지 + 핵심 키워드 + N분에] 정리.
```
- 길이: 한국어 약 80~110자 / 영어 약 130~160자.
- 첫 문장에서 "나도 그래!" 반응 유도, 둘째 문장에 핵심 키워드.

### TLDR (첫 단락 / JSON-LD `abstract` → Featured Snippet 후보)
```
[정의 한 줄, 명확한 단어로]
```
- 길이: 40~95자. 호기심형 X, **사실·정의형 O** (구글이 답으로 그대로 쓸 수 있게 정확히).
- title/desc는 호기심, **tldr는 사실** — 역할 분리.

## ✅ 작성 후 체크리스트 (커밋 전 전부 만족)

### Title
- [ ] 앞부분에 호기심·공감·질문형 훅이 있는가?
- [ ] em dash(—) 뒤에 핵심 키워드 2~3개가 자연스럽게 들어갔는가?
- [ ] 모바일에서 잘리지 않는 길이인가? (ko ~32자 / en ~60자)
- [ ] "완벽 정리/총정리/Ultimate Guide"로 끝나지 않는가?
- [ ] 본문 H1과 100% 중복되지 않는가?

### Description
- [ ] 첫 문장이 "나도 그래!" 반응을 끄는 공감·호기심인가?
- [ ] 둘째 문장에 핵심 쿼리 단어가 들어갔는가?
- [ ] 모바일 SERP에서 잘리지 않는 길이인가?

### TLDR
- [ ] 정의·사실로 시작하는가? (호기심형 X)
- [ ] 핵심 키워드를 자연어 문장으로 풀어 썼는가?
- [ ] 95자 이내인가?

## 🔥 좋은 예시 vs 나쁜 예시

### 버튼 포지션 (button-position)
| 구분 | ❌ 정보형 | ✅ 하이브리드 |
|---|---|---|
| Title | 홀덤 버튼 포지션 완벽 가이드 — 장점 총정리 | **약패도 버튼이면 들어가는 이유 — 홀덤 버튼 포지션·블라인드 스틸** |
| Desc | 홀덤 버튼 포지션의 장점을 정리했습니다… | **버튼이 좋은 자리라는 말만 들었지 왜인지는 모르셨죠?** 정보 우위·블라인드 스틸·핸드 레인지까지 실전 기준으로 정리했습니다. |
| TLDR | (사실형) | 버튼(BTN)은 플랍 이후 항상 마지막에 행동하는 자리로, 정보 우위 덕분에 레인지 확장·블라인드 스틸·팟 컨트롤이 가장 쉽습니다. |

### 팟 오즈 (pot-odds)
| 구분 | ❌ 정보형 | ✅ 하이브리드 |
|---|---|---|
| Title | 홀덤 팟 오즈 계산법 — 완벽 정리 | **콜이 손해인지 1초 만에 아는 법 — 홀덤 팟 오즈 계산** |
| Desc | 팟 오즈 계산법을 정리했습니다… | **드로우 잡고 콜할까 폴드할까 매번 헷갈리셨죠?** 팟 오즈와 4·2 룰로 콜의 손익을 즉석에서 판단하는 법을 예시로 정리했습니다. |

## 💡 호기심 훅 패턴 라이브러리

### A. 결과 의문형 (가장 강력)
- "분명 OO인데 왜 ××했을까?"
- "같은 OO인데 왜 내가 졌지?"
- "왜 매번 OO에서 칩이 빠질까?"

### B. 손해·실수 자극형
- "OO 자리에서 항상 손해 보는 이유"
- "초보가 OO에서 가장 많이 잃는 한 가지"

### C. 비밀·반전형
- "사실 OO은 자리 탓이 아닙니다"
- "프로들은 OO에서 다르게 행동합니다 — 그 이유"

### D. 시간 압축형
- "5분이면 OO이 끝납니다 — 핵심만"

### E. 숫자형
- "OO에서 가장 많이 틀리는 3가지"

> ⚠️ B/C/E는 강력하지만 과장 금지. 본문이 약속을 못 지키면 이탈률↑ → 순위↓.

## 🌐 다국어 번역 원칙 (기계 번역 절대 금지)

### 🚫 AI 기계 번역 절대 금지

**번역기·AI가 단어를 1:1로 치환하는 방식은 이 사이트에서 절대 허용하지 않는다.**

포커는 언어마다 용어 관행이 다르다. 기계 번역은 맥락을 모르기 때문에:
- 영어 그대로 쓰는 게 자연스러운 용어를 억지로 번역하거나
- 이미 해당 언어 포커 커뮤니티에서 굳어진 표현 대신 낯선 단어를 쓴다.

그 결과: 원어민 독자가 읽자마자 "번역 티"를 느낌 → 신뢰도 하락 → 이탈 → 순위 하락.

### 홀덤 용어 번역 지침

**원본이 영어이므로, 번역이 어색하거나 마땅한 현지어가 없으면 영어 그대로 쓰면 된다.**
피해야 하는 건 "번역을 못 하는 것"이 아니라, **사전적 의미만 보고 맥락을 무시한 기계적 치환**이다.

| 상황 | 원칙 |
|---|---|
| 영어 용어가 해당 언어 커뮤니티에서도 그대로 쓰임 | 영어 유지 (억지 번역 X) |
| 현지 번역어가 굳어진 경우 | 현지 관용어 사용 |
| 두 가지 모두 혼용되는 경우 | 자연스러운 쪽 선택 또는 현지어 + 영어 병기 |
| 마땅한 번역어가 없거나 번역하면 어색한 경우 | **영어 그대로** + 필요하면 문장 안에서 뜻 풀어주기 |

**핵심 기준 — 이 질문 하나로 판단**
> "이 언어를 쓰는 포커 플레이어가 읽었을 때 자연스럽게 이해되는가?"
> YES → 통과. NO → 다시 쓴다. 영어가 더 자연스러우면 영어가 정답.

> 번역 전 반드시 해당 언어의 포커 관련 유튜브·커뮤니티에서 **실제로 어떻게 표현하는지 확인**할 것.

### 톤 가이드 (언어별)

| 언어 | 톤 | 비고 |
|---|---|---|
| ko | 친근체 + 1인칭 경험 | 원본/검수 |
| en | Casual, 직설적, 실전 위주 | 마스터 |
| es / pt-BR | 따뜻하고 친근, LATAM 중립 | |
| de / ru | 논리적·정확, 데이터 강조 | |
| ja | 정중체, 단계별 친절 | |
| fr / it | 우아하지만 명료 | |
| tr / pl / vi / th / id / hi / uk | 현지 구어체, 직역 티 제거 | |
| ar | 정중체 + RTL 레이아웃 | 후순위 |

- 한국어 호기심 훅을 그대로 직역하면 어색 → **각 언어 SERP 톤으로 현지화**.
- 슬러그는 영어 고정, 메타·본문만 번역.

## ⚠️ 절대 금지

1. 호기심형 제목을 키워드만으로 단독 교체 ❌ (호기심 살리고 키워드만 보강).
2. 본문 친근체를 메타에 맞춰 깎는 것 ❌ (본문은 친구처럼, 메타는 자극적으로).
3. 모든 글이 같은 패턴으로 보이는 것 ❌ (A~E 골고루).
4. 본문 약속을 못 지키는 과장 ❌.
5. AI 정형문 인트로 ❌ ("오늘은 ~알아보겠습니다", "본 포스팅에서는…" 등).
6. **사전적 의미만 보고 맥락을 무시한 기계적 번역 그대로 발행 ❌** — 번역이 어색하거나 마땅한 현지어가 없으면 영어를 유지하는 게 낫다. 문제는 "번역 못 하는 것"이 아니라 "맥락 없는 1:1 치환"이다.
7. 포커 용어를 사전 뜻 그대로 1:1 치환 ❌ — "이 언어 포커 플레이어에게 자연스러운가?"가 유일한 기준.

## 🔁 GSC 기반 카피 개선 워크플로우 (발행 후)

1. GSC 실적 → 페이지(URL) 필터 → 쿼리 탭 확인.
2. 노출↑·클릭 적은 쿼리 추출 → 검색 의도 파악.
3. 현재 title/desc 진단 (의도 단어 누락 / 호기심 부족 / 둘 다).
4. 하이브리드 공식으로 재작성 (호기심 훅 + 누락 키워드).
5. 첫 화면에 "한 줄 정리" 박스 추가 (의도 즉답 + 체류↑).
6. `updated` 갱신 → 빌드 검증 → 커밋.
7. 2주 뒤 GSC 재확인 → CTR·평균 순위 추적.

## 📊 성과 추적 기준

| 지표 | 최소 | 목표 |
|---|---|---|
| 평균 순위 | 10위 이내 (1페이지) | **5위 이내** |
| CTR (평균 7~8위) | 3%+ | **5%+** |
| 클릭 수 (월간) | 노출의 4%+ | **노출의 6%+** |

- CTR 5%+ → 카피 유지. CTR 3% 미만 + 순위 7위 이내 → **카피만 바꿔도 즉시 효과**.

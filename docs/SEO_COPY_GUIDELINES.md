# 🎯 SEO 카피라이팅 가이드 — 호기심 훅 + 키워드 하이브리드

> 홀덤마스터(holdemmaster.com) 모든 포스트의 `seoTitle` / `desc` 작성 원칙.
> 본 문서의 원칙은 `.cursorrules`와 동기화됨.
> 마지막 갱신: 2026-05-19

---

## 🚨 최우선 — 신규·대규모 수정 포스팅 전 필수 (절대 생략 금지)

**포스팅할 때는 항상 타깃 키워드로 구글 검색 → 상위 10개 경쟁 글을 분석한 뒤 → 그보다 경쟁력 있는 콘텐츠를 만든다.**

메타·본문·이미지를 쓰기 **전에** 아래를 끝내야 한다. 이 단계 없이 글 쓰기 시작 = 금지.

| 순서 | 할 일 | 산출물 |
|:----:|------|--------|
| 1 | **타깃 키워드 1~3개** 확정 (GSC 쿼리·로드맵·사용자 의도) | 예: `홀덤 체크 뜻`, `스몰블라인드 역할` |
| 2 | 구글에서 **키워드 그대로** 검색 (시크릿 모드 권장) | SERP 스크린샷 또는 URL 목록 |
| 3 | **상위 10개** (광고 제외, 블로그·가이드 위주) URL 열어보기 | 경쟁 글 10개 체크리스트 |
| 4 | 각 글에서 **H2 목차·분량·FAQ·표·이미지·실전 예시** 유무 비교 | “우리가 더 줄 것” 3가지 |
| 5 | **차별 포인트** 정하기 — E-E-A-T 실전 1~2건, 빠진 질문, 더 나은 표/인포 | 글 설계 1페이지 |
| 6 | 그 설계대로 **초안 작성** → 메타(호기심 훅) → 내부링크 | 배포 |

### 경쟁 분석 시 반드시 적을 것 (메모)

- 상위 글 **평균 H2 개수** / **대략 분량** (짧으면 우리가 깊게, 길면 핵심만 압축)
- **빠진 질문** (댓글·People Also Ask·FAQ에 있는데 본문에 없는 것)
- **우리만 넣을 것**: 홀덤펍 실전 에피소드, 한국어 초보 맞춤, 클러스터 내부링크
- **이기기 어려운 것**: 대형 사이트·동영상만 있는 주제 → 롱테일·각도 변경

### 한 줄 원칙

> **“정보를 더 많이”가 아니라 “검색 의도를 더 정확히 + 클릭·체류를 더 잘” 충족하는 글.**

## 🧭 핵심 철학

**구글 검색 결과의 제목·설명 = 유튜브 썸네일.**
영상 자체가 좋아도 썸네일이 평범하면 클릭이 안 일어난다. 검색 결과도 동일하다.

- 한국어 SERP의 경쟁 글들은 대부분 “OO 뜻 / OO 정리 / OO 가이드” 같은 **딱딱한 정보형**이다.
- 그 안에서 우리 글이 살아남으려면 **호기심·공감·감정을 자극하는 훅**으로 시선을 잡아야 한다.
- 동시에 구글 알고리즘이 “이 글이 검색어와 관련 있다”고 판단하도록 **핵심 키워드**도 들어가야 한다.
- 즉, **호기심 훅 + 핵심 키워드 = 하이브리드** 가 정답.

---

## 📐 검증된 공식

### Title (`seoTitle`)

```
[호기심·자극·공감 훅] — [핵심 키워드 명사 2~3개]
```

- 길이: **한국어 약 28~34자** (모바일 SERP에서 잘리지 않는 안전선)
- 구분자는 **em dash (—)** 통일 (가독성 + 시선 끌림 동시)
- 끝 단어는 **명사로 종결** (“정리/가이드” 같은 동사형 마무리 자제)
- “완벽 정리”, “총정리”, “모든 것” 같은 평범한 표현 **사용 금지**

### Description (`desc`)

```
[첫 문장: 공감·호기심 한 문장 — 검색자의 고민·증상]. [둘째 문장: 무엇을 알 수 있는지 + 핵심 키워드 + 시간(N분에)] 정리했습니다.
```

- 길이: **한국어 약 80~110자**
- 첫 문장에서 **검색자의 “나도 그래!” 반응**을 끌어내야 한다.
- 둘째 문장에서 **무엇을 어떤 깊이로 알 수 있는지** + 키워드를 자연스럽게 노출.

### TLDR (`tldr`) — JSON-LD `abstract`로 노출 (Featured Snippet 후보)

```
[정의 한 줄, 핵심 단어 굵게 인식되도록 명확한 단어 사용]
```

- 길이: **40~95자**
- 호기심형 X, **사실·정의형 O** (이건 구글이 직접 답으로 쓸 수 있는 텍스트라 정확해야 함)
- title/desc는 호기심, **tldr는 사실** — 역할 분리 기억할 것

## ✅ 작성 후 체크리스트

`seoTitle`과 `desc` 작성 후 아래를 모두 만족해야 커밋:

### Title 체크
- [ ] 첫 부분에 호기심·공감·질문형 훅이 있는가?
- [ ] em dash(`—`) 뒤에 핵심 키워드 2~3개가 자연스럽게 들어갔는가?
- [ ] 32자 안팎으로 잘리지 않을 길이인가?
- [ ] “완벽 정리/총정리” 같은 평범한 단어로 끝나지 않는가?
- [ ] 본문 H1(`title`)과 100% 중복되지 않는가? (검색용·본문용은 서로 보완 관계)

### Description 체크
- [ ] 첫 문장이 검색자의 “나도 그래!” 반응을 끌어내는 호기심·공감인가?
- [ ] 둘째 문장에 GSC 핵심 쿼리 단어가 들어갔는가?
- [ ] “정리”, “설명”, “가이드” 같은 단어가 마지막에 자연스럽게 끝나는가?
- [ ] 95자 안팎으로 모바일 SERP에서 잘리지 않는가?

### TLDR 체크
- [ ] 정의·사실로 시작하는가? (호기심형 X)
- [ ] 핵심 키워드를 자연어 문장으로 풀어 썼는가?
- [ ] 95자 이내인가?

## 🔥 좋은 예시 vs 나쁜 예시 사례집

### 1) `holdem-blind-meaning` (블라인드 뜻)

| 구분 | ❌ 나쁨 (정보형) | ✅ 좋음 (하이브리드) |
|------|-------------------|----------------------|
| Title | 홀덤 블라인드 뜻 — 스몰·빅블라인드 차이와 금액 정리 | **매판마다 칩이 빠지는 이유 — 홀덤 블라인드 뜻과 SB·BB 차이** |
| Desc | 홀덤 블라인드 뜻과 스몰블라인드·빅블라인드 차이를 초보 눈높이로 정리했습니다… | **블라인드 자리에 앉으면 매판 칩이 줄어드는 느낌, 사실 자리 탓이 아닙니다.** 홀덤 블라인드 뜻부터 SB·BB 금액·차이, 블라인드 자리에서 이기는 실전 전략까지 5분에 정리했습니다. |
| TLDR | (불변) | 블라인드는 카드를 받기 전 의무로 내는 강제 베팅 — 딜러 왼쪽이 스몰블라인드(SB, 빅블라인드의 절반), 그다음이 빅블라인드(BB, 기준 금액)입니다. |

### 2) `holdem-tiebreak-rules` (타이브레이크)

| 구분 | ❌ 나쁨 (정보형) | ✅ 좋음 (하이브리드) |
|------|-------------------|----------------------|
| Title | 홀덤 같은 족보 승자 결정법 — 키커·스플릿 팟 규칙 정리 | **같은 원페어인데 왜 내가 졌지? — 홀덤 키커·스플릿 팟 규칙** |
| Desc | 홀덤에서 같은 족보일 때 누가 이기는지 결정하는 규칙을 정리했습니다… | **분명 같은 원페어였는데 졌습니다 — 키커 한 장이 승부를 가른 거예요.** 홀덤에서 같은 족보일 때 누가 이기는지, 키커 5장 비교·스플릿 팟·무늬 적용 여부까지 사례로 정리했습니다. |
| TLDR | (불변) | 같은 족보면 키커를 1·2·3·4·5장째 순서로 비교, 5장까지 모두 같으면 스플릿 팟으로 팟을 나눕니다. 무늬(스페이드 등)는 홀덤 승자 결정에 영향이 없습니다. |

## 💡 호기심 훅 패턴 라이브러리

다음 패턴을 글 주제에 맞게 변형해 사용한다.

### A. 결과 의문형 (가장 강력)
- "분명 OO인데 왜 ××했을까?"
- "OO처럼 보이지만 사실은 ××입니다"
- "같은 OO인데 왜 내가 졌지?"
- "왜 매번 OO에서 칩이 빠질까?"

### B. 손해·실수 자극형
- "OO 자리에서 항상 손해 보는 이유"
- "초보자가 OO에서 가장 많이 잃는 한 가지"
- "이거 모르면 매판 OO원씩 새고 있는 거예요"

### C. 비밀·반전형
- "사실 OO은 자리 탓이 아닙니다"
- "프로들은 OO에서 다르게 행동합니다 — 그 이유"
- "OO 단 한 끗으로 승부가 갈렸어요"

### D. 시간 압축형
- "5분이면 OO이 끝납니다 — 핵심만"
- "이 한 페이지로 OO 마스터하기"

### E. 숫자형
- "OO에서 가장 많이 틀리는 3가지"
- "초보 95%가 모르는 OO 규칙"

> ⚠️ B/C/E는 강력하지만 과장 금지. 본문이 약속을 지키지 못하면 이탈률 ↑ → 순위 ↓

---

## ⚠️ 절대 금지 사항

1. **CTR 잘 나오는 호기심형 제목을 정보형으로 단독 교체** ❌
   - 키워드 부족하다고 호기심을 통째로 들어내지 말 것
   - 항상 “호기심 살리고 키워드만 보강” 방식으로
2. **본문 톤(친근체)을 메타에 맞춰 깎는 것** ❌
   - 본문은 친구처럼, 메타는 자극적으로 — 둘은 별개
3. **모든 글이 비슷한 패턴으로 보이는 것** ❌
   - 호기심 패턴 A~E를 골고루 섞어 다양성 유지
4. **자극이 본문 약속을 못 지키는 과장** ❌
   - 검색 클릭 후 “낚였다”고 느끼면 즉시 이탈 → 순위 하락 신호

## 🔁 GSC 데이터 기반 카피 개선 워크플로우

기존 글을 다듬을 때는 항상 다음 순서로:

1. **GSC `실적 → 페이지: 해당 URL` 필터** → **쿼리 탭** 확인
2. **노출 ↑ / 클릭 적은 쿼리** 추출 → 검색 의도 파악
3. **현재 `seoTitle` / `desc` 진단**: 의도 단어 누락 / 호기심 부족 / 둘 다
4. **하이브리드 공식**으로 다시 작성 (호기심 훅 + 누락 키워드)
5. **본문 첫 화면**에 “한 줄 정리” 박스 추가 (의도에 즉답 + 체류 향상)
6. **`updated` 날짜 갱신** + 빌드 검증 → 커밋 → 푸시
7. **2주 뒤 GSC 재확인** → CTR / 평균 순위 변화 추적

## 📊 성과 추적 기준

각 글별 목표:

| 지표 | 최소 | 목표 |
|------|------|------|
| 평균 순위 | 10위 이내 (1페이지) | **5위 이내** |
| CTR (평균 7~8위 기준) | 3% 이상 | **5% 이상** |
| 클릭 수 (월간) | 노출의 4% 이상 | **노출의 6%+** |

CTR 5% 이상이 나오면 그 카피는 **유지**가 정답.
CTR 3% 미만 + 평균 순위 7위 이내면 **카피만 바꿔도 즉시 효과**.
